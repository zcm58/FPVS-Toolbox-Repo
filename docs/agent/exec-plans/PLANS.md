# Plans

Execution plans live in `docs/agent/exec-plans/`.

- `active/`: read before changing covered areas.
- `future/`: approved ideas not currently being implemented.
- `completed/`: placeholder only; remove completed plans instead of retaining
  implementation logs. Use git history only when the user asks for completed
  plan context.
- `tech-debt-tracker.md`: known debt not yet promoted to active work.

Small one-off changes do not need an execution plan.
