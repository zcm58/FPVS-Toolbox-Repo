# Docs Knowledge Base

This directory is the repo-local system of record for FPVS Toolbox. Agents should
start with `AGENTS.md`, `ARCHITECTURE.md`, and `docs/agent-index.md`, then use
this map to choose the narrowest deeper source of truth.

## Directory Map

- `architecture/`: durable architecture maps, ownership boundaries, and behavior contracts.
- `design-docs/`: product and engineering beliefs that guide future design choices.
- `exec-plans/`: active, future, completed, and technical-debt plans.
- `generated/`: generated reference artifacts. Do not edit generated files by hand.
- `product-specs/`: durable product behavior and workflow specifications.
- `quality/`: test-selection and verification-gate guidance.
- `references/`: external or copied reference material useful to agents.
- `reviews/`: pre-ship and review checklists.
- `statistics/`: statistical methods and analysis reference.
- `tools/`: tool-specific user and implementation docs.

## Root Knowledge Files

- `DESIGN.md`: design-system and UX guidance.
- `FRONTEND.md`: PySide6 frontend implementation guidance.
- `PLANS.md`: plan index and planning rules.
- `PRODUCT_SENSE.md`: product goals and user workflow priorities.
- `QUALITY_SCORE.md`: current quality posture and gaps.
- `RELIABILITY.md`: reliability and verification expectations.
- `SECURITY.md`: security and privacy boundaries.

Keep this page compact. Add details to focused docs, not here.
